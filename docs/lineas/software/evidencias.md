# 2. Evidencias de la línea de Software

| CE | Evidencia del programa (artefacto + condición de logro) | Código |
|---|---|---|
| **CE021** Ingeniería de Requerimientos | Documento de requerimientos completo, consistente y validado con stakeholders (incluye requerimientos funcionales y no funcionales) | **CE0211** |
|  | Prototipos navegables alineados a los requerimientos y validados con usuarios | **CE0212** |
|  | Arquitectura del sistema definida mediante vistas, componentes y decisiones técnicas coherentes con los requerimientos | **CE0213** |
|  | Modelado del sistema mediante diagramas UML estructurales y de comportamiento consistentes con la arquitectura | **CE0214** |
| **CE022** Ingeniería de la Información | Modelo de datos conceptual, lógico y físico consistente con los requerimientos del sistema | **CE0221** |
|  | Consultas y manipulación de datos mediante SQL correctas, eficientes y alineadas al modelo de datos | **CE0222** |
|  | Programación de base de datos (procedimientos, triggers, funciones) funcional y consistente con la lógica del sistema | **CE0223** |
|  | Base de datos segura, optimizada y administrada correctamente en entorno operativo | **CE0224** |
| **CE023** Programación | Aplicación de consola funcional que resuelve un problema estructurado | **CE023a** |
|  | Aplicación de escritorio funcional con arquitectura en capas y persistencia de datos | **CE023b** |
|  | Aplicación web basada en arquitectura MVC funcional e integrada con base de datos | **CE023c** |
|  | Aplicación web full-stack segura e integrada (backend + frontend) | **CE023d** |
|  | Sistema basado en microservicios con comunicación, resiliencia y consistencia implementadas | **CE023e** |
|  | Aplicación móvil integrada al backend y alineada al sistema desarrollado | **CE023f** |
| **CE024** Calidad de Software | Pruebas automatizadas que validan el comportamiento del sistema (unitarias, integración, E2E) | **CE0241** |
|  | Pipeline de integración y entrega continua (CI/CD) funcional que automatiza validación y despliegue | **CE0242** |
|  | Gestión técnica del desarrollo evidenciada mediante decisiones, control de deuda técnica y revisión de código | **CE0243** |
|  | Auditoría de calidad del software con métricas, evaluación técnica y plan de evolución del sistema | **CE0244** |
