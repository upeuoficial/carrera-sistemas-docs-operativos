# RÚBRICA NIVEL 3 — CE023 (Programación) — EPE

## CONTEXTO DE EVALUACIÓN

Se evalúa sobre:
- Sistema funcional completo integrado (web + distribuido + móvil)
- Proveniente de DIST + MOV + integración del PI

## CRITERIOS DE EVALUACIÓN

| Criterio                              | Excelente (18–20)                                                                 | Bueno (15–17)                                                      | Regular (13–14)                              | Deficiente (<13)                  |
|---------------------------------------|-----------------------------------------------------------------------------------|---------------------------------------------------------------------|-----------------------------------------------|------------------------------------|
| Arquitectura e integración del sistema| Arquitectura claramente definida e implementada; integración completa entre componentes (backend, microservicios, frontend, móvil) sin inconsistencias | Arquitectura implementada con integración funcional, presenta leves desacoples o inconsistencias menores | Arquitectura parcialmente implementada; integración incompleta o con problemas visibles | No existe arquitectura clara; componentes desconectados |
| Funcionalidad del sistema             | Todas las funcionalidades definidas operan correctamente en todos los módulos del sistema | Funcionalidad mayormente operativa, con fallos menores no críticos  | Funcionalidades incompletas o inconsistentes   | El sistema no cumple con los requerimientos funcionales |
| Integración entre capas y servicios   | Comunicación fluida entre frontend, backend y servicios; manejo adecuado de APIs y contratos | Integración funcional con pequeños problemas de comunicación        | Integración parcial o inconsistente            | Componentes no integrados          |
| Calidad del código                    | Código limpio, modular, desacoplado, con uso adecuado de patrones de diseño         | Código estructurado con algunos problemas de organización           | Código poco modular o con duplicidad           | Código desordenado y difícil de mantener |
| Desempeño y comportamiento del sistema| Sistema responde eficientemente bajo diferentes condiciones; buen manejo de recursos | Rendimiento aceptable con leves degradaciones                       | Problemas de rendimiento evidentes             | Sistema inestable o lento           |
| Escalabilidad y diseño técnico        | Sistema preparado para escalar (microservicios, desacoplamiento, configuraciones externas) | Diseño escalable parcial                                             | Diseño rígido                                   | Sistema no escalable                |

## VALIDACIÓN

- Está alineada a CE023
- Evalúa el producto real (sistema)
- No mezcla calidad (eso va en CE024)
- Es defendible en acreditación
- Es observable y medible

**No evalúa:**
- Pruebas
- CI/CD
- Métricas

(Estos aspectos se evalúan en CE024)
