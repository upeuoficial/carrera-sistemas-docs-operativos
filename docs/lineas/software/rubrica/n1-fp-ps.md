# RÚBRICA NIVEL 1 — FP (Fundamentos de Programación, Ciclo 1) — PS

## CONTEXTO DE EVALUACIÓN

Se evalúa sobre:
- Producto final del curso: aplicación de consola completa (menú, condicionales, bucles, subprogramas, archivos)
- Evidencia: CE023a (consola)

## PRINCIPIO

- Evalúa la capacidad de construir una pieza funcional de software a nivel inicial.

## CRITERIOS DE EVALUACIÓN

| Criterio                        | Excelente (17–20)                                                                 | Bueno (13–16)                                   | Regular (10–12)                        | Deficiente (0–9)                |
|---------------------------------|-----------------------------------------------------------------------------------|-------------------------------------------------|-----------------------------------------|-------------------------------|
| Funcionalidad                   | El programa resuelve correctamente el problema planteado y cumple todos los requisitos | Funcionalidad mayormente operativa con detalles menores | Funcionalidad incompleta o con errores | El programa no cumple el objetivo |
| Estructura y modularidad        | Uso adecuado de subprogramas, código organizado y reutilizable                    | Estructura funcional con oportunidades de mejora | Estructura poco modular o desordenada   | Código desorganizado o sin subprogramas |
| Manejo de entradas y salidas    | Entradas y salidas gestionadas correctamente, validación de datos                  | Manejo funcional con detalles menores            | Manejo limitado o con errores           | No gestiona entradas/salidas correctamente |
| Uso de archivos                 | Implementa correctamente operaciones con archivos (lectura/escritura)              | Uso funcional con detalles menores               | Uso limitado o con errores              | No implementa manejo de archivos |
| Documentación y estilo          | Código bien documentado y con estilo claro                                         | Documentación y estilo adecuados                 | Documentación limitada                  | Sin documentación ni estilo      |

## ESTRUCTURA DE LA RÚBRICA

- Funcionalidad
- Estructura
- Entradas/Salidas
- Archivos
- Documentación

## VALIDACIÓN

- ✔ Alineada a FP (c1)
- ✔ Evalúa pieza funcional de software
- ✔ Es defendible y observable

## FRASE CLAVE

> “El primer paso para ser programador profesional es construir piezas funcionales, claras y bien estructuradas.”
