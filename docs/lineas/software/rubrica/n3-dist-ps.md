# RÚBRICA NIVEL 3 — DIST (Distribuidas, Ciclo 5) — PS

## CONTEXTO DE EVALUACIÓN

Se evalúa sobre:
- Producto final del sistema integrado: sistema distribuido (microservicios, comunicación, resiliencia, despliegue)
- Evidencia: CE023e (microservicios)

## PRINCIPIO

- Evalúa la capacidad de construir, integrar y operar un sistema distribuido profesional como parte de un sistema completo.

## CRITERIOS DE EVALUACIÓN

| Criterio                        | Excelente (18–20)                                                                 | Bueno (15–17)                                   | Regular (13–14)                        | Deficiente (<13)                |
|---------------------------------|-----------------------------------------------------------------------------------|-------------------------------------------------|-----------------------------------------|-------------------------------|
| Arquitectura distribuida        | Arquitectura de microservicios clara, escalable y bien definida                    | Arquitectura funcional con oportunidades de mejora | Arquitectura poco clara o rígida        | No hay arquitectura definida     |
| Comunicación y resiliencia      | Comunicación fluida y manejo de fallos entre servicios                             | Comunicación funcional con detalles menores      | Comunicación limitada o con errores     | No hay comunicación ni manejo de fallos |
| Despliegue y operación          | Sistema desplegado y operando en entorno definido                                 | Despliegue funcional con detalles menores        | Despliegue parcial o con problemas      | No es posible desplegar el sistema      |
| Integración con el sistema      | Sistema distribuido correctamente integrado al sistema principal                   | Integración funcional con detalles menores       | Integración parcial o con problemas     | No hay integración funcional     |
| Documentación y estilo          | Código y arquitectura bien documentados                                            | Documentación y estilo adecuados                 | Documentación limitada                  | Sin documentación ni estilo      |

## ESTRUCTURA DE LA RÚBRICA

- Arquitectura distribuida
- Comunicación
- Despliegue
- Integración
- Documentación

## VALIDACIÓN

- ✔ Alineada a DIST (c5) y EPE (N3)
- ✔ Evalúa integración profesional de sistemas distribuidos
- ✔ Es defendible y observable

## FRASE CLAVE

> “El dominio profesional de sistemas distribuidos se valida cuando los microservicios funcionan, se comunican y escalan como parte de un sistema completo.”
