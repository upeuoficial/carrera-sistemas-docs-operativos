# RÚBRICA NIVEL 2 — LP1 (Lenguajes de Programación 1, Ciclo 3) — PI

## CONTEXTO DE EVALUACIÓN

Se evalúa sobre:
- Producto PI: aplicación web MVC integrada, alineada a los requerimientos y datos del sistema funcional del ciclo 3
- Evidencia: CE023c (web MVC)

## PRINCIPIO

- Evalúa la capacidad de construir e integrar una aplicación web funcional en el PI de ciclo 3.

## CRITERIOS DE EVALUACIÓN

| Criterio                        | Excelente (18–20)                                                                 | Bueno (15–17)                                   | Regular (13–14)                        | Deficiente (<13)                |
|---------------------------------|-----------------------------------------------------------------------------------|-------------------------------------------------|-----------------------------------------|-------------------------------|
| Integración de componentes web  | Todos los módulos y vistas funcionan e interactúan correctamente                   | Integración funcional con detalles menores       | Integración parcial o con fallos        | Componentes desconectados        |
| Funcionalidad del sistema       | El sistema cumple todos los requerimientos funcionales definidos                   | Funcionalidad mayormente operativa               | Funcionalidades incompletas o inconsistentes | El sistema no cumple los requerimientos |
| Arquitectura MVC                | Arquitectura MVC definida y aplicada; separación de capas y responsabilidades      | Arquitectura funcional con limitaciones          | Arquitectura poco clara o rígida         | No hay arquitectura definida     |
| Integración con datos           | Integración correcta con la base de datos y operaciones CRUD                      | Integración funcional con detalles menores       | Integración parcial o con problemas      | No hay integración con datos     |
| Documentación                   | Código y arquitectura bien documentados                                            | Documentación y estilo adecuados                 | Documentación limitada                  | Sin documentación ni estilo      |

## ESTRUCTURA DE LA RÚBRICA

- Integración web
- Funcionalidad
- Arquitectura MVC
- Integración con datos
- Documentación

## VALIDACIÓN

- ✔ Alineada a LP1 (c3)
- ✔ Evalúa integración de software web en el PI
- ✔ Es defendible y observable

## FRASE CLAVE

> “El desarrollo profesional de software inicia con la integración de aplicaciones web funcionales y bien estructuradas.”
