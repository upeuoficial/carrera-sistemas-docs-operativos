# RÚBRICA NIVEL 1 — POO (Programación Orientada a Objetos, Ciclo 2) — PS

## CONTEXTO DE EVALUACIÓN

Se evalúa sobre:
- Producto final del curso: aplicación de escritorio completa (modelo POO, CRUD, GUI, persistencia básica)
- Evidencia: CE023b (desktop)

## PRINCIPIO

- Evalúa la capacidad de construir una pieza funcional de software orientada a objetos a nivel inicial.

## CRITERIOS DE EVALUACIÓN

| Criterio                        | Excelente (17–20)                                                                 | Bueno (13–16)                                   | Regular (10–12)                        | Deficiente (0–9)                |
|---------------------------------|-----------------------------------------------------------------------------------|-------------------------------------------------|-----------------------------------------|-------------------------------|
| Funcionalidad                   | El programa cumple todos los requisitos y opera correctamente                      | Funcionalidad mayormente operativa con detalles menores | Funcionalidad incompleta o con errores | El programa no cumple el objetivo |
| Modelado orientado a objetos    | Clases, objetos y relaciones correctamente definidos y aplicados                   | Modelado funcional con oportunidades de mejora   | Modelado incompleto o con errores       | No aplica POO o es incorrecto    |
| Interfaz de usuario (GUI)       | Interfaz funcional, intuitiva y alineada a los requisitos                          | Interfaz funcional con detalles menores          | Interfaz limitada o poco usable         | No hay interfaz o es inusable    |
| Persistencia de datos           | Implementa correctamente almacenamiento y recuperación de datos                    | Persistencia funcional con detalles menores      | Persistencia limitada o con errores     | No implementa persistencia       |
| Documentación y estilo          | Código bien documentado y con estilo claro                                         | Documentación y estilo adecuados                 | Documentación limitada                  | Sin documentación ni estilo      |

## ESTRUCTURA DE LA RÚBRICA

- Funcionalidad
- Modelado POO
- Interfaz
- Persistencia
- Documentación

## VALIDACIÓN

- ✔ Alineada a POO (c2)
- ✔ Evalúa pieza funcional orientada a objetos
- ✔ Es defendible y observable

## FRASE CLAVE

> “El dominio de la programación orientada a objetos inicia con la construcción de aplicaciones funcionales, bien modeladas y documentadas.”
