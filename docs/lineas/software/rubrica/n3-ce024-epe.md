# RÚBRICA NIVEL 3 — CE024 (Calidad de Software) — EPE

## CONTEXTO DE EVALUACIÓN

Se evalúa sobre:
- Sistema completo desarrollado (CE023) + validado + operado + evaluado
- Incluye: pruebas automatizadas, pipeline CI/CD, gestión técnica (IS1), auditoría y mejora (IS2)

## PRINCIPIO DE ESTA RÚBRICA

- NO evalúa si el sistema funciona (eso es CE023)
- Evalúa qué tan confiable, controlado y mejorable es el sistema

## CRITERIOS DE EVALUACIÓN

| Criterio                        | Excelente (18–20)                                                                 | Bueno (15–17)                                   | Regular (13–14)                        | Deficiente (<13)                |
|---------------------------------|-----------------------------------------------------------------------------------|-------------------------------------------------|-----------------------------------------|-------------------------------|
| Cobertura y calidad de pruebas  | Pruebas unitarias, integración y/o E2E cubren los componentes críticos; detectan errores reales | Pruebas funcionales con cobertura parcial       | Pruebas limitadas o poco efectivas     | No hay pruebas o no validan el sistema |
| Automatización (CI/CD)          | Pipeline automatiza pruebas, build y despliegue de forma consistente               | Pipeline funcional con limitaciones             | Automatización parcial                  | No existe automatización         |
| Uso de métricas y evaluación    | Se utilizan métricas (rendimiento, errores, uso) para evaluar el sistema           | Métricas básicas implementadas                  | Métricas limitadas o poco utilizadas    | No se evalúa el sistema con métricas  |
| Gestión técnica del desarrollo  | Evidencia clara de decisiones técnicas, control de deuda técnica y revisión de código | Gestión técnica presente pero parcial           | Gestión técnica limitada                | No hay gestión técnica            |
| Auditoría técnica del sistema   | Evaluación estructurada del sistema (arquitectura, código, datos, calidad)         | Auditoría general adecuada                      | Auditoría superficial                   | No hay auditoría                  |
| Propuesta de mejora continua    | Plan de evolución basado en evidencia (métricas, fallos, riesgos)                  | Propuesta de mejora general                     | Propuesta limitada                      | No hay mejora propuesta            |

## ESTRUCTURA DE LA RÚBRICA

- Pruebas
- CI/CD
- Métricas
- Gestión técnica
- Auditoría
- Mejora continua

## VALIDACIÓN (CLAVE)

- ✔ Alineada a CE024
- ✔ Integra IS1 + PDS + IS2
- ✔ No duplica CE023
- ✔ Evalúa calidad real del software
- ✔ Es defendible ante acreditación

### DIFERENCIA CLAVE CON CE023

| CE023                | CE024                |
|----------------------|----------------------|
| Construye el sistema | Evalúa el sistema    |
| Código               | Calidad              |
| Funcionalidad        | Confiabilidad        |
| Arquitectura         | Mejora continua      |

**Frase clave:**

> “La calidad del software no se evidencia en su funcionamiento, sino en su capacidad de ser validado, medido, controlado y mejorado.”
