# RÚBRICA NIVEL 2 — CE023 (Programación) — PI

## CONTEXTO DE EVALUACIÓN

Se evalúa sobre:
- Sistema funcional desarrollado e integrado en el PI de ciclo 3 (LP1) o ciclo 4 (LP2)
- Incluye: integración de componentes, funcionalidad, arquitectura básica, calidad de código y despliegue funcional

## PRINCIPIO

- Evalúa la capacidad de construir un sistema funcional e integrado a nivel de ciclo, no solo piezas aisladas.

## CRITERIOS DE EVALUACIÓN

| Criterio                        | Excelente (18–20)                                                                 | Bueno (15–17)                                   | Regular (13–14)                        | Deficiente (<13)                |
|---------------------------------|-----------------------------------------------------------------------------------|-------------------------------------------------|-----------------------------------------|-------------------------------|
| Integración de componentes      | Todos los módulos y componentes funcionan e interactúan correctamente              | Integración funcional con detalles menores       | Integración parcial o con fallos        | Componentes desconectados        |
| Funcionalidad del sistema       | El sistema cumple todos los requerimientos funcionales definidos                   | Funcionalidad mayormente operativa               | Funcionalidades incompletas o inconsistentes | El sistema no cumple los requerimientos |
| Arquitectura y diseño técnico   | Arquitectura básica definida y aplicada; separación de capas y responsabilidades   | Arquitectura funcional con limitaciones          | Arquitectura poco clara o rígida         | No hay arquitectura definida     |
| Calidad del código              | Código estructurado, modular y documentado                                        | Código funcional con problemas menores           | Código poco modular o con duplicidad     | Código desordenado o difícil de mantener |
| Despliegue y operación básica   | El sistema es desplegable y operable en el entorno definido                       | Despliegue funcional con detalles menores        | Despliegue parcial o con problemas       | No es posible desplegar el sistema      |

## ESTRUCTURA DE LA RÚBRICA

- Integración de componentes
- Funcionalidad
- Arquitectura
- Código
- Despliegue

## VALIDACIÓN

- ✔ Alineada a LP1 (c3) y LP2 (c4)
- ✔ Evalúa integración y funcionalidad a nivel de ciclo
- ✔ No duplica EPE (nivel 3)
- ✔ Es defendible y observable

## FRASE CLAVE

> “Un programador profesional no solo construye piezas, sino sistemas funcionales e integrados.”
