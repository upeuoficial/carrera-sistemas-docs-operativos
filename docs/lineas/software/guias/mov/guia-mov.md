# 📘 Guía de Proyecto — PS  
Desarrollo de Aplicaciones Móviles (MOV) — Ciclo 6

---

## 🎯 1. Propósito

La presente guía orienta el desarrollo del Proyecto Sello (PS) del curso de Desarrollo de Aplicaciones Móviles, en el marco del enfoque por competencias del programa.

👉 Este proyecto se desarrolla bajo la guía general:

- [Guía de Proyectos — PS / PI / EPE](../proyectos.md)

El proyecto MOV desarrolla una **aplicación móvil integrada a servicios backend**, como parte del sistema completo del estudiante.

---

## 🧠 2. Enfoque

> Se evalúa la capacidad de construir aplicaciones móviles funcionales integradas a servicios.

El estudiante desarrolla una solución progresivamente:

- Definición del problema  
- Diseño de interfaz y flujo  
- Integración con backend  
- Implementación móvil  
- Validación  

Los entregables representan **evidencias de competencias**, no fases.

---

## 🔍 3. Naturaleza del proyecto

El proyecto:

- Resuelve un problema real o contextualizado  
- Se implementa como aplicación móvil  
- Se conecta a servicios backend (propios o del PI/DIST)  
- Se desarrolla en equipo  

> ✔ Debe existir integración real con servicios

## 🎯 11. Nivel esperado

PS — Nivel profesional inicial:

- App funcional completa  
- Integración real con servicios  
- Flujo coherente  
- Experiencia de usuario adecuada  

---

## ⚠️ 12. Consideraciones

- Trabajo en equipo obligatorio  
- Uso de repositorio (Git)  
- Integración con backend obligatoria  
- Coherencia entre UI y lógica  

🚫 No se aceptan:

- apps sin backend  
- pantallas sin funcionalidad  
- datos simulados sin justificación  
- apps no integradas  

---

## 🏁 Mensaje final

> El estudiante desarrolla aplicaciones móviles conectadas a sistemas reales, integrando frontend, servicios y experiencia de usuario como parte de un sistema completo.
