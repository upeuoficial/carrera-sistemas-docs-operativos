# 📘 Guía de Proyecto — PS  
Pruebas y Despliegue de Software (PDS) — Ciclo 7

---

## 🎯 1. Propósito

La presente guía orienta el desarrollo del Proyecto Sello (PS) del curso de Pruebas y Despliegue de Software, en el marco del enfoque por competencias del programa.

👉 Este proyecto se desarrolla bajo la guía general:

- [Guía de Proyectos — PS / PI / EPE](../proyectos.md)

El proyecto PDS se enfoca en la **validación, automatización y despliegue del sistema**, asegurando su calidad técnica.

---

## 🧠 2. Enfoque

> Se evalúa la capacidad de validar, automatizar y desplegar un sistema de software.

El estudiante trabaja sobre un sistema existente:

- Identificación de funcionalidades  
- Diseño de pruebas  
- Automatización  
- Integración continua (CI/CD)  
- Despliegue  

Los entregables representan **evidencias de competencias**, no fases.

---

## 🔍 3. Naturaleza del proyecto

El proyecto:

- Se basa en un sistema previamente desarrollado  
- Evalúa su calidad técnica  
- Automatiza procesos de desarrollo  
- Simula un entorno real de despliegue  

> ✔ Se prioriza validación y confiabilidad del sistema

## 🎯 11. Nivel esperado

PS — Nivel profesional:

- Sistema probado  
- Automatización funcional  
- Despliegue real  
- Evaluación técnica clara  

---

## ⚠️ 12. Consideraciones

- Uso de repositorio obligatorio  
- Uso de CI/CD obligatorio  
- Evidencia de pruebas obligatoria  
- Documentación técnica clara  

🚫 No se aceptan:

- pruebas manuales sin evidencia  
- sistema sin pipeline  
- despliegue inexistente  
- sin métricas
