# 📘 Guía de Proyecto — PI1  
Proyecto Integrador — Ciclo 3 (IR, BD1, LP1)

---

## 🎯 1. Propósito

La presente guía orienta el desarrollo del Proyecto Integrador (PI1) del ciclo 3, integrando los cursos:

- Ingeniería de Requerimientos (IR)  
- Base de Datos I (BD1)  
- Lenguajes de Programación I (LP1)  

👉 Este proyecto se desarrolla bajo la guía general:

- [Guía de Proyectos — PS / PI / EPE](../proyectos.md)

El PI1 consolida la construcción de un **sistema web funcional con base de datos**.

---

## 🧠 2. Enfoque

> Se evalúa la capacidad de integrar requerimientos, datos e implementación en un sistema funcional.

El estudiante desarrolla un sistema progresivamente:

- Definición del problema  
- Diseño de la solución  
- Modelado de datos  
- Implementación web (MVC)  
- Integración completa  

Los entregables representan **evidencias de competencias**, no fases.

---

## 🔍 3. Naturaleza del proyecto

El proyecto:

- Resuelve un problema real o contextualizado  
- Se implementa como sistema web  
- Integra frontend, backend y base de datos  
- Se desarrolla en equipo  

> ✔ No se aceptan componentes desconectados

## 🧩 10. Tipo de sistema

---

## 🎯 11. Nivel esperado

PI — Nivel intermedio:

- Sistema funcional completo  
- Integración real entre capas  
- Consistencia entre requerimientos, datos y código  
- Base sólida para EPE  

---

## ⚠️ 12. Consideraciones

- Trabajo en equipo obligatorio  
- Uso de repositorio (Git)  
- Coherencia entre IR → BD → LP  
- Validación continua  

🚫 No se aceptan:

- requerimientos no implementados  
- base de datos aislada  
- frontend sin backend  
- sistema sin integración  

---

## 🏁 Mensaje final

> El estudiante integra requerimientos, datos y programación para construir un sistema funcional, estableciendo la base para arquitecturas más complejas en ciclos posteriores.
