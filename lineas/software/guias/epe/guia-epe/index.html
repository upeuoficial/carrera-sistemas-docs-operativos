    📘 Guía de Proyecto — EPE¶

    Evaluación del Perfil de Egreso — Línea Software

    🎯 1. Propósito¶

    La presente guía orienta la Evaluación del Perfil de Egreso (EPE) en la línea de software, como etapa final del proceso formativo.

    • Guía de Proyectos — PS / PI / EPE

    El EPE evalúa la capacidad del estudiante para integrar, validar y sustentar un sistema de software a nivel profesional.

    El sistema evaluado en el EPE debe cumplir con las condiciones definidas en la línea de software: Ver alcance del proyecto

    🧠 2. Enfoque¶

    El EPE no evalúa cursos, evalúa el logro del perfil de egreso.

    Se evalúa:

    • Integración de competencias
    • Coherencia del sistema
    • Calidad técnica
    • Capacidad de sustentación

    🎯 Se evalúa integración, no acumulación de entregables.

    🔍 3. Naturaleza del EPE¶

    El EPE no implica necesariamente el desarrollo de un nuevo proyecto desde cero.

    Consiste en la:

    • Integración
    • Consolidación
    • Validación

    de evidencias desarrolladas en:

    • Proyectos Sello (PS)
    • Proyectos Integradores (PI)
    • Proyecto nuevo en caso corresponda

    Los productos del EPE corresponden a artefactos integrados de nivel profesional, no a entregables aislados.

    🧩 4. Sistema a evaluar¶

    El estudiante presenta un sistema:

    • Funcional
    • Integrado
    • Coherente con el problema
    • Técnicamente justificado

    Puede ser:

    • Web (MVC / full-stack)
    • Microservicios
    • Aplicación móvil
    • Escritorio (según contexto)

    ✔ No se fuerza tecnología
    ✔ Se evalúa la solución

    🧾 5. Brief integrado¶

    El proceso inicia con la consolidación del problema del proyecto.

    • Plantilla de Brief

    Debe incluir:

    • Problema
    • Objetivos de la solución
    • Alcance
    • Contexto y Stakeholders

    📌 Hito obligatorio:

    • Aprobación del Brief
    • Evaluación interna

    👉 El brief aprobado es requisito para continuar

    🧪 6. Evaluación del EPE¶

    La evaluación se realiza en dos niveles:

    🔹 Evaluación interna¶

    • Validación del Brief
    • Consistencia del sistema
    • Coherencia de la solución

    🔹 Evaluación con jurado externo¶

    Evalúa:

    • Nivel profesional
    • Integración del sistema
    • Calidad técnica
    • Sustentación

    📦 7. Evidencias del EPE¶

    El sistema debe integrar evidencias de:

    Competencia Evidencia
    CE021 Requerimientos y arquitectura
    CE022 Datos y persistencia
    CE023 Implementación
    CE024 Calidad y evaluación
    CE0217 Sustentación

    • Evaluación EPE

    • CE021 — Nivel 3

    • CE022 — Nivel 3
    • CE023 — Nivel 3
    • CE024 — Nivel 3
    • Sustentación

    🧭 8. Proceso del EPE¶

    El EPE se desarrolla en dos fases:

    🔹 Fase 1 — Validación interna¶

    • Brief aprobado
    • Arquitectura definida
    • Diseño validado

    🔹 Fase 2 — Evaluación final¶

    • Sistema completo
    • Evidencia de calidad
    • Sustentación ante jurado

    🎤 9. Sustentación¶

    Incluye:

    • Video pitch (1–3 min)
    • Demo del sistema
    • Explicación técnica
    • Justificación de decisiones
    • Respuesta a preguntas

    Evaluada mediante CE0217

    🎯 10. Nivel esperado¶

    EPE — Nivel profesional:

    • Sistema completo e integrado
    • Arquitectura implementada
    • Calidad técnica demostrable
    • Sustentación sólida

    ⚠️ 11. Consideraciones¶

    • El EPE no implica desarrollar un nuevo sistema
    • Se basa en evidencias o entregables (E1, E2, E3, E4) y E5
    • El sistema debe ser funcional
    • Debe existir coherencia total con la línea y con las otras líneas de la carrera
    • La sustentación es obligatoria

    🚫 No se aceptan:

    • componentes desconectados
    • sistemas incompletos
    • soluciones sin justificación
    • evidencia no verificable

    🏁 Mensaje final¶

    El EPE representa la demostración del perfil de egreso, donde el estudiante integra sus competencias para construir, validar y sustentar un sistema de software en un contexto real.
